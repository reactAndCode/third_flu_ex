# 요구사항 정의서 (PRD)
## Product Requirements Document

**프로젝트명**: 윤상민 운동 관리 앱
**버전**: 1.1.0
**작성일**: 2024-11-20
**플랫폼**: Android, iOS (Web), Web

---

## 1. 프로젝트 개요

### 1.1 배경
개인 운동 기록을 체계적으로 관리하고 추적할 수 있는 모바일/웹 애플리케이션의 필요성

### 1.2 목적
- 일일 운동 루틴을 쉽게 기록하고 조회
- 운동 데이터를 날짜별로 관리
- 크로스 플랫폼 접근성 제공 (Android, iOS, Web)

### 1.3 범위
- **포함**: 운동 기록 CRUD, 사용자 인증, 날짜별 조회, 웹 배포
- **제외**: 운동 분석 그래프, 소셜 기능, 운동 추천 알고리즘

## 3. 기능 요구사항

### 3.1 사용자 인증 (필수)

#### FR-1: 회원가입
- **설명**: 이메일/비밀번호로 신규 계정 생성
- **입력**: 이메일, 비밀번호, 비밀번호 확인
- **출력**: 회원가입 성공/실패 메시지
- **제약사항**:
  - 이메일 형식 유효성 검증
  - 비밀번호 최소 6자 이상
  - 중복 이메일 방지

#### FR-2: 로그인
- **설명**: 등록된 계정으로 로그인
- **입력**: 이메일, 비밀번호
- **출력**: 로그인 성공 시 홈 화면 이동
- **제약사항**:
  - 유효한 자격증명만 허용
  - 세션 자동 유지

#### FR-3: 로그아웃
- **설명**: 현재 세션 종료
- **입력**: 로그아웃 버튼 클릭
- **출력**: 로그인 화면으로 이동

### 3.2 운동 관리 (필수)

#### FR-4: 운동 추가
- **설명**: 새로운 운동 기록 생성
- **입력**:
  - 운동명 (필수)
  - 중량 (선택)
  - 횟수 (선택)
  - 세트 (선택)
  - 운동부위 (선택)
  - 운동방법상세 (선택)
  - 날짜 (현재 선택된 날짜)
- **출력**: 운동 목록에 새 항목 표시
- **제약사항**: 운동명은 필수 입력

#### FR-5: 운동 조회
- **설명**: 특정 날짜의 운동 기록 조회
- **입력**: 날짜 선택
- **출력**: 해당 날짜의 운동 목록
- **제약사항**: 날짜별로 그룹화하여 표시

#### FR-6: 운동 수정
- **설명**: 기존 운동 기록 수정
- **입력**: 수정할 운동 항목, 새로운 값
- **출력**: 업데이트된 운동 정보
- **제약사항**: 본인이 작성한 운동만 수정 가능

#### FR-7: 운동 삭제
- **설명**: 운동 기록 삭제
- **입력**: 삭제할 운동 항목
- **출력**: 삭제 확인 메시지
- **제약사항**:
  - 삭제 전 확인 다이얼로그 표시
  - 본인이 작성한 운동만 삭제 가능

#### FR-8: 운동 상세보기
- **설명**: 운동 기록의 상세 정보 조회
- **입력**: 운동명 클릭
- **출력**: 운동 상세 화면 표시
- **제약사항**: 모든 필드 표시 (운동방법상세 포함)

### 3.3 날짜 탐색 (필수)

#### FR-9: 이전/다음 날짜 이동
- **설명**: 날짜를 전후로 이동하며 운동 기록 조회
- **입력**: 이전/다음 버튼 클릭
- **출력**: 해당 날짜의 운동 목록
- **제약사항**: 날짜 및 요일을 한국어로 표시

## 4. 비기능 요구사항

### 4.1 성능
- **NFR-1**: 운동 목록 로딩 시간 2초 이내
- **NFR-2**: 운동 추가/수정/삭제 응답 시간 1초 이내

### 4.2 보안
- **NFR-3**: Supabase Row Level Security (RLS) 적용
- **NFR-4**: 사용자별 데이터 격리
- **NFR-5**: HTTPS 통신 사용

### 4.3 가용성
- **NFR-6**: 99% 이상 가동률 (Vercel 기준)
- **NFR-7**: 오프라인 모드 미지원 (온라인 전용)

### 4.4 사용성
- **NFR-8**: 직관적인 UI/UX
- **NFR-9**: 한국어 지원
- **NFR-10**: 반응형 웹 디자인 (모바일 최적화)

### 4.5 호환성
- **NFR-11**: Android 5.0 (API 21) 이상
- **NFR-12**: iOS 11 이상
- **NFR-13**: 주요 웹 브라우저 지원 (Chrome, Safari, Firefox, Edge)

## 5. 사용자 시나리오

### 5.1 신규 사용자 회원가입 및 운동 기록
```
1. 앱 실행
2. 회원가입 화면에서 이메일/비밀번호 입력
3. 회원가입 완료 후 자동 로그인
4. 홈 화면에서 "+" 버튼 클릭
5. 운동 정보 입력 (예: 벤치프레스, 60kg, 10회, 3세트, 가슴)
6. 저장 버튼 클릭
7. 운동 목록에 새 항목 표시
```

### 5.2 기존 사용자 로그인 및 기록 조회
```
1. 앱 실행
2. 자동 로그인 (세션 유지)
3. 홈 화면에서 오늘 날짜의 운동 목록 확인
4. 이전 버튼 클릭하여 어제 운동 기록 조회
5. 운동명 클릭하여 상세 정보 확인
```

### 5.3 운동 기록 수정
```
1. 홈 화면에서 수정할 운동 항목 길게 누르기
2. 수정 다이얼로그에서 정보 변경
3. 저장 버튼 클릭
4. 업데이트된 정보 확인
```

## 7. 제약사항 및 가정

### 7.1 제약사항
- 인터넷 연결 필수 (오프라인 모드 미지원)
- Supabase 무료 플랜 제한 (월 500MB 데이터베이스, 2GB 대역폭)
- Flutter 웹은 PWA로 배포되지 않음

### 7.2 가정
- 사용자는 하루에 최대 20개의 운동 기록
- 운동방법상세는 500자 이내
- 사용자는 기본적인 스마트폰 사용 능력 보유
