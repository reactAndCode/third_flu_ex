## 3. 화면별 상세 설계

### SCR-001: 로그인 화면

#### 3.1.1 화면 목적
- 기존 사용자가 이메일/비밀번호로 로그인
- 회원가입 화면으로 이동

#### 3.1.2 UI 구성요소

```
┌─────────────────────────────────┐
│         My Awesome              │ ← AppBar (핑크 하트 아이콘)
├─────────────────────────────────┤
│                                 │
│         로그인                  │ ← Title (큰 글자)
│                                 │
│  ┌───────────────────────────┐ │
│  │ 이메일                    │ │ ← Email TextField
│  └───────────────────────────┘ │
│                                 │
│  ┌───────────────────────────┐ │
│  │ 비밀번호                  │ │ ← Password TextField
│  └───────────────────────────┘ │
│                                 │
│  ┌───────────────────────────┐ │
│  │       로그인              │ │ ← Login Button (파란색)
│  └───────────────────────────┘ │
│                                 │
│  계정이 없으신가요? 회원가입    │ ← Signup Link
│                                 │
└─────────────────────────────────┘
```

#### 3.1.3 필드 정의

| 필드명 | 타입 | 필수 | 검증 규칙 |
|--------|------|------|-----------|
| 이메일 | TextField | Y | 이메일 형식 |
| 비밀번호 | TextField (obscure) | Y | 6자 이상 |

#### 3.1.4 동작 정의

| 동작 | 트리거 | 결과 |
|------|--------|------|
| 로그인 | 로그인 버튼 클릭 | - 성공: 홈 화면 이동<br>- 실패: 에러 메시지 표시 |
| 회원가입 이동 | 회원가입 링크 클릭 | 회원가입 화면으로 이동 |

#### 3.1.5 에러 처리

| 에러 상황 | 메시지 |
|-----------|--------|
| 이메일 미입력 | "이메일을 입력해주세요" |
| 비밀번호 미입력 | "비밀번호를 입력해주세요" |
| 로그인 실패 | "이메일 또는 비밀번호가 올바르지 않습니다" |
| 네트워크 오류 | "네트워크 연결을 확인해주세요" |

### SCR-002: 회원가입 화면

#### 3.2.1 화면 목적
- 신규 사용자 계정 생성
- 이메일/비밀번호 기반 인증

#### 3.2.2 UI 구성요소

```
┌─────────────────────────────────┐
│    ←    회원가입                │ ← AppBar (뒤로가기)
├─────────────────────────────────┤
│                                 │
│         회원가입                │ ← Title
│                                 │
│  ┌───────────────────────────┐ │
│  │ 이메일                    │ │ ← Email TextField
│  └───────────────────────────┘ │
│                                 │
│  ┌───────────────────────────┐ │
│  │ 비밀번호                  │ │ ← Password TextField
│  └───────────────────────────┘ │
│                                 │
│  ┌───────────────────────────┐ │
│  │ 비밀번호 확인             │ │ ← Confirm Password TextField
│  └───────────────────────────┘ │
│                                 │
│  ┌───────────────────────────┐ │
│  │       가입하기            │ │ ← Signup Button (파란색)
│  └───────────────────────────┘ │
│                                 │
└─────────────────────────────────┘
```

#### 3.2.3 필드 정의

| 필드명 | 타입 | 필수 | 검증 규칙 |
|--------|------|------|-----------|
| 이메일 | TextField | Y | 이메일 형식 |
| 비밀번호 | TextField (obscure) | Y | 6자 이상 |
| 비밀번호 확인 | TextField (obscure) | Y | 비밀번호와 일치 |

#### 3.2.4 동작 정의

| 동작 | 트리거 | 결과 |
|------|--------|------|
| 회원가입 | 가입하기 버튼 클릭 | - 성공: 자동 로그인 후 홈 화면 이동<br>- 실패: 에러 메시지 표시 |
| 뒤로가기 | 뒤로가기 버튼 클릭 | 로그인 화면으로 이동 |

#### 3.2.5 에러 처리

| 에러 상황 | 메시지 |
|-----------|--------|
| 이메일 미입력 | "이메일을 입력해주세요" |
| 이메일 형식 오류 | "올바른 이메일 형식이 아닙니다" |
| 비밀번호 미입력 | "비밀번호를 입력해주세요" |
| 비밀번호 6자 미만 | "비밀번호는 6자 이상이어야 합니다" |
| 비밀번호 불일치 | "비밀번호가 일치하지 않습니다" |
| 중복 이메일 | "이미 사용 중인 이메일입니다" |

### SCR-003: 홈 화면 (운동 목록)

#### 3.3.1 화면 목적
- 선택된 날짜의 운동 기록 표시
- 운동 추가/수정/삭제
- 날짜 탐색

#### 3.3.2 UI 구성요소

```
┌─────────────────────────────────┐
│  ❤  My Awesome       로그아웃  │ ← AppBar
├─────────────────────────────────┤
│  나의 운동 내역                 │ ← Subtitle (20pt)
│                                 │
│  ← 2024년 11월 20일 수요일  →  │ ← Date Navigation
│                                 │
│  ┌─────────────────────────┐   │
│  │ [B] 벤치프레스          │   │ ← Workout Item 1
│  │ 가슴 | 60kg×10 | 3세트  │   │   (클릭 가능 - 파란색 밑줄)
│  │                    ⋮    │   │   (길게 누르기 - 수정/삭제)
│  └─────────────────────────┘   │
│                                 │
│  ┌─────────────────────────┐   │
│  │ [S] 스쿼트              │   │ ← Workout Item 2
│  │ 하체 | 80kg×8 | 4세트   │   │
│  │                    ⋮    │   │
│  └─────────────────────────┘   │
│                                 │
│                          [+]    │ ← Floating Action Button
└─────────────────────────────────┘
```

#### 3.3.3 컴포넌트 상세

**날짜 네비게이션**
- 형식: `YYYY년 MM월 DD일 요일` (한국어)
- 좌우 화살표로 날짜 이동
- 클릭 시 날짜 선택 다이얼로그 (선택 사항)

**운동 항목 (WorkoutListItem)**
```
┌─────────────────────────────┐
│ [첫글자] 운동명              │ ← 32x32 동그라미 + 운동명
│ 부위 | 중량×횟수 | 세트     │ ← 운동 정보 한 줄
│                        ⋮     │ ← 메뉴 아이콘
└─────────────────────────────┘
```

**추가 버튼 다이얼로그**
```
┌─────────────────────────────┐
│  운동 추가                   │
├─────────────────────────────┤
│  운동명: _______________    │
│  중량:   _______________    │
│  횟수:   _______________    │
│  세트:   _______________    │
│  운동부위: _____________    │
│  운동방법상세: _________    │
│                             │
│      [취소]    [저장]       │
└─────────────────────────────┘
```

#### 3.3.4 동작 정의

| 동작 | 트리거 | 결과 |
|------|--------|------|
| 운동 추가 | FAB (+) 클릭 | 운동 추가 다이얼로그 표시 |
| 운동 저장 | 다이얼로그에서 저장 버튼 | 운동 목록에 추가, 다이얼로그 닫기 |
| 운동 상세보기 | 운동명 클릭 | 운동 상세 화면으로 이동 |
| 운동 수정 | 항목 길게 누르기 → 수정 | 수정 다이얼로그 표시 |
| 운동 삭제 | 항목 길게 누르기 → 삭제 | 확인 다이얼로그 후 삭제 |
| 이전 날짜 | ← 버튼 클릭 | 전날 운동 목록 로드 |
| 다음 날짜 | → 버튼 클릭 | 다음날 운동 목록 로드 |
| 로그아웃 | 로그아웃 버튼 클릭 | 로그인 화면으로 이동 |

#### 3.3.5 상태 표시

| 상태 | UI |
|------|-----|
| 로딩 중 | 중앙에 CircularProgressIndicator |
| 데이터 없음 | "운동 기록이 없습니다" 메시지 |
| 에러 발생 | 에러 메시지 + 재시도 버튼 |

### SCR-004: 운동 상세 화면

#### 3.4.1 화면 목적
- 운동 기록의 모든 정보 상세 표시
- 운동방법상세 내용 확인

#### 3.4.2 UI 구성요소

```
┌─────────────────────────────────┐
│    ←    운동 상세                │ ← AppBar (뒤로가기)
├─────────────────────────────────┤
│                                 │
│  운동명                         │ ← Label
│  벤치프레스                     │ ← Value (큰 글자)
│                                 │
│  ─────────────────────────────  │
│                                 │
│  날짜                           │
│  2024년 11월 20일 수요일        │
│                                 │
│  ─────────────────────────────  │
│                                 │
│  운동부위                       │
│  가슴                           │
│                                 │
│  ─────────────────────────────  │
│                                 │
│  중량                           │
│  60kg                           │
│                                 │
│  ─────────────────────────────  │
│                                 │
│  횟수                           │
│  10회                           │
│                                 │
│  ─────────────────────────────  │
│                                 │
│  세트                           │
│  3세트                          │
│                                 │
│  ─────────────────────────────  │
│                                 │
│  운동방법상세                   │
│  어깨를 뒤로 당기고 가슴을      │
│  펴서 바를 가슴 중앙까지        │
│  내린다...                      │
│                                 │
└─────────────────────────────────┘
```

#### 3.4.3 필드 표시

| 필드명 | 형식 | 비고 |
|--------|------|------|
| 운동명 | Text | 큰 글자 (24pt) |
| 날짜 | YYYY년 MM월 DD일 요일 | 한국어 |
| 운동부위 | Text | - |
| 중량 | Text | 단위 포함 |
| 횟수 | Text | 단위 포함 |
| 세트 | Text | 단위 포함 |
| 운동방법상세 | Text | 다중 라인, 최대 500자 |

#### 3.4.4 동작 정의

| 동작 | 트리거 | 결과 |
|------|--------|------|
| 뒤로가기 | 뒤로가기 버튼 클릭 | 홈 화면으로 돌아가기 |

## 5. 반응형 디자인

### 5.1 모바일 (< 600px)
- 단일 컬럼 레이아웃
- 전체 너비 버튼
- 스택 네비게이션

### 5.2 태블릿 (600px - 1200px)
- 2컬럼 레이아웃 (목록 + 상세)
- 사이드바 네비게이션

### 5.3 데스크톱 (> 1200px)
- 최대 너비 제한 (1200px)
- 중앙 정렬

## 7. 접근성

- **색상 대비**: WCAG AA 준수 (4.5:1)
- **터치 영역**: 최소 48x48 픽셀
- **폰트 크기**: 최소 14pt
- **키보드 네비게이션**: 지원 안 함 (모바일 우선)
